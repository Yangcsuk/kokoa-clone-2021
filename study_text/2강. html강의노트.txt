2 Learning HTML
2.1 Our First HTML File
  - 파일명과 폴더명은 소문자(lower case)로 작성해야 한다.
2.2~9 Tag, Attributes and ID
  - <!DOCTYPE html> 이 코드는 없어도 되지만 html문서라고 브라우저에게 알려주는 기능을 수행한다.
  - 어떤 태그는 id라는 arrtribute를 가질 수 있다.ex) image, paragraph. header, link...
  - atrribute 값은 항상 "" 큰 따옴표 안에 작성한다.
  - 반대로 src(source)라는 attribute는 모든 태그가 가질 수 있지 않다.
  - 코드 자체에 의미가 부여된 semantic 태그를 잊지 말자.ex) header, navigation, footer...
  - semantic 태그로 코드를 작성 하는 것은 매우 중요하다. 작성된 코드들이 훨씬 더 보기 좋다.
  - header, main, footer, navigation, hgroup 등 <>속 태그들은 전부 container이다. 전부 div 태그로 대체 할 수 있다.
2.10 Semantic 
  - Semantic 뜻 : 문서를 보기만 해도 그 의미를 짐작할 수 있다.(header, main, footer 등)
  - div와 header, main, footer 의 차이점 : 기능은 동일하나 프로그램을 짜는 사람들이 보기 편하게 만들어준다.
  - 모든 태그를 암기 할 필요는 없다. 필요할 때마다 문서를 찾아 적용하면 된다.
